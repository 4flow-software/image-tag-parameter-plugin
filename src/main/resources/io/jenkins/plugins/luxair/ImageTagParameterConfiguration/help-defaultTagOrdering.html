Enables administrators to define the default tag ordering for any newly created parameter.

Natural Ordering ... same Ordering as the tags had in prior versions
Reverse Natural Ordering ... the reversed original ordering
Descending Versions ... tries to parse the tags as versions and arrange them in descending order.
Ascending Versions ... attempts to parse the tags to a version and order them ascending.

WARNING:
Be careful when using versioned sorting. This method is most effective with semver or Maven version-compliant ImageTags but may not perform optimally with other schemas!
The conversion of tags to versions may fail in specific cases, leading to sorting issues and potentially resulting in an empty selection list. In such situations, users will receive a notification indicating a parameter misconfiguration.
